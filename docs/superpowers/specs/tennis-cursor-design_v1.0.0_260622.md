# Tennis 전역 테니스공 커서 설계

## 목적

`tennis.fedshot911.com`의 홈페이지, Privacy, Contact, Racket Finder에서 마우스 커서를 모두 기존 테니스공 이미지로 통일한다.

## 현재 원인

테니스공 커서 변수와 전역 선택자가 홈페이지 전용 클래스인 `.tennis-home-root`와 `.tennis-home-page`에만 적용되어 있다. Privacy 등 하위 페이지의 `body`에는 해당 클래스가 없어 기본 커서가 표시된다. Racket Finder 전용 CSS에는 `pointer`, `not-allowed` 같은 개별 커서 규칙도 있다.

## 변경 설계

- 공통 파일 `styles.v1.1.1.css`의 최상위 범위에 기존 테니스공 커서 이미지 변수를 둔다.
- `html`, `body`, `body *`에 동일한 테니스공 커서를 전역 적용한다.
- 나중에 불러오는 하위 페이지 CSS의 `pointer`와 `not-allowed`보다 전역 규칙이 우선하도록 커서 속성에만 `!important`를 사용한다.
- 모든 HTML의 공통 CSS 주소에 같은 캐시 갱신값을 붙여 배포 후 이전 CSS가 남는 문제를 방지한다.
- 커서 이미지, 화면 디자인, 문구, 기능 로직은 변경하지 않는다.

## 적용 범위

- 홈페이지
- Privacy
- Contact
- Racket Finder
- 링크, 버튼, 입력 요소, 비활성 버튼을 포함한 모든 화면 요소

## 검증

- 정적 검사로 모든 HTML이 갱신된 공통 CSS를 불러오는지 확인한다.
- CSS 우선순위를 검사해 개별 `cursor` 선언이 전역 규칙을 덮어쓸 수 없는지 확인한다.
- 로컬 브라우저에서 각 페이지의 본문, 링크, 버튼, 비활성 버튼의 계산된 `cursor` 값을 확인한다.
- 기존 커서 이미지 파일이 실제로 존재하고 로드되는지 확인한다.

## 주의 사항

비활성 버튼에서도 금지 표시 커서 대신 테니스공이 보인다. 비활성 여부는 기존 투명도 표현으로 구분하며, 이는 모든 커서를 통일해 달라는 승인 범위에 따른다.
