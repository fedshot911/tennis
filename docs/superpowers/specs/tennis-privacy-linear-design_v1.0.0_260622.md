# Tennis Privacy Linear형 디자인 설계

## 목적

`tennis.fedshot911.com/privacy/`를 Linear Privacy Policy의 단정한 문서 구조로 재구성하되, 현재 tennis Privacy의 아이보리·딥그린·검정 색상과 모든 개인정보 처리 문구를 유지한다.

## 참고 기준

- 공식 레퍼런스: `https://linear.app/privacy`
- 참고 요소: 큰 문서 제목, 시행일, 짧은 소개, 번호형 목차, 좁은 단일 본문 열, 명확한 제목 단계, 넓은 여백, 섹션 구분선.
- 복제하지 않는 요소: Linear 로고, 색상, 글꼴, 문구, 푸터, 서비스 탐색 구조.

## 확정 범위

- 현재 Privacy에 적힌 저장 항목, 저장 목적, 제3자 제공, 보관 기간, 인프라, 안전성, 문의 문구를 삭제하거나 의미 변경하지 않는다.
- 헤더, 공통 푸터, 테니스공 커서는 유지한다.
- Privacy HTML과 Privacy 전용 CSS만 변경한다.
- 새 이미지, JavaScript, 라이브러리, 외부 글꼴은 추가하지 않는다.

## 화면 구조

### 1. 문서 헤더

- 작은 `Privacy` 라벨을 유지한다.
- `개인정보 처리 안내`를 페이지의 가장 큰 제목으로 표시한다.
- 제목 아래에 `시행일 및 마지막 수정일: 2026년 6월 21일`을 배치한다.
- 현재 소개 문단을 그대로 이어서 배치한다.

### 2. 상단 요약

- 기존 3개 카드형 요약을 제거한다.
- 저장 목적, 저장 위치, 문의 정보를 테두리 없는 3행 정의 목록으로 표시한다.
- 항목명은 딥그린 또는 진한 본문색, 값은 현재 보조 본문색을 사용한다.

### 3. 페이지 목차

- `이 페이지에서` 제목 아래 번호형 내부 링크를 배치한다.
- 목차 항목은 다음 순서를 사용한다.
  1. 저장하는 정보
  2. 저장하지 않는 정보
  3. 이용 목적
  4. 보관 기간과 삭제
  5. 제3자 제공
  6. 처리 위탁과 인프라
  7. 안전성 확보 조치
  8. 문의와 요청
- 링크는 각 본문 섹션 ID로 이동한다.

### 4. 본문

- 기존 2열 카드 그리드를 없애고 모든 섹션을 하나의 문서 열에 순서대로 배치한다.
- 각 섹션은 작은 영문 라벨, 한국어 `h2`, 기존 본문 순서로 구성한다.
- 섹션 사이에는 얇은 구분선과 충분한 상하 여백을 둔다.
- 현재 카드 배경, 둥근 테두리, 그림자는 본문에서 제거한다.

### 5. 저장 정보 표

- 표의 행과 문구는 모두 유지한다.
- 외곽 카드 느낌을 줄이고, 헤더와 행 사이의 얇은 구분선 중심으로 단순화한다.
- 모바일에서는 기존과 같이 표 영역만 가로 스크롤을 허용한다.

### 6. 문의

- 현재 이메일 주소와 안내 문구를 유지한다.
- 검정색 알약형 버튼 대신 밑줄 또는 얇은 하단선이 있는 텍스트 링크로 표시한다.

## 시각 체계

- 배경: 현재 아이보리 계열 배경 유지.
- 본문: 현재 검정 `#181a1f` 계열 유지.
- 강조: 현재 딥그린 `#205843` 계열 유지.
- 보조 본문: 현재 회색 계열 유지.
- 본문 최대 폭은 약 `760px`에서 `820px` 사이로 제한해 긴 문장을 읽기 쉽게 한다.
- 제목은 현재 시스템 글꼴을 사용하고 Linear처럼 크기와 여백의 대비로 위계를 만든다.

## 반응형

- 데스크톱과 모바일 모두 단일 열 구조를 유지한다.
- 모바일에서 제목 크기, 섹션 여백, 목차 간격만 축소한다.
- 본문과 목차는 화면 폭 안에 들어오며, 저장 정보 표만 독립적으로 가로 스크롤한다.

## 접근성

- 목차는 `nav`와 설명 가능한 `aria-label`을 사용한다.
- 모든 목차 링크는 실제 섹션 ID와 일치한다.
- 제목 단계는 `h1` 다음 `h2` 순서로 유지한다.
- 이메일 링크와 목차 링크의 키보드 포커스가 보이도록 한다.
- 색상만으로 정보의 의미를 구분하지 않는다.

## 검증

- 기존 Privacy 본문 핵심 문구와 표의 행이 모두 남아 있는지 정적 검사한다.
- 목차 8개 링크와 대상 ID가 정확히 연결되는지 검사한다.
- 로컬 브라우저에서 데스크톱과 390 × 844 모바일 화면을 확인한다.
- 목차 링크를 눌러 해당 섹션으로 이동하는지 확인한다.
- 표 외 영역에 가로 스크롤, 겹침, 잘림이 없는지 확인한다.
- 헤더, 공통 푸터, 테니스공 커서가 유지되는지 확인한다.
- 관련 console error/warning과 이미지 로드 오류가 없는지 확인한다.

## 오류 처리

- JavaScript를 추가하지 않으므로 별도 실행 오류 경로는 없다.
- 내부 링크 대상이 누락되면 정적 회귀 검사에서 실패하도록 한다.

## 변경 제외

- 개인정보 처리 문구의 법률 검토나 정책 내용 추가는 이번 디자인 작업에 포함하지 않는다.
- Contact, 홈페이지, Racket Finder 본문 디자인은 변경하지 않는다.
