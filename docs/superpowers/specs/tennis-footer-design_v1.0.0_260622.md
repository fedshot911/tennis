# Tennis 공통 푸터 설계

## 목적

`tennis.fedshot911.com`의 홈페이지, Privacy, Contact, Racket Finder 하단을 동일한 구조와 모양으로 통일한다.

## 확정 구성

- 좌측에는 기존 fedshot911 심볼 이미지만 표시한다.
- 심볼은 tennis 홈페이지 `/`로 이동하는 링크로 사용한다.
- 우측에는 `Privacy · Contact` 링크만 표시한다.
- 현재 보고 있는 페이지와 관계없이 네 페이지에서 같은 링크 두 개를 유지한다.
- `Racket Finder`, `fedshot911 archive`, `fedshot911 tennis` 텍스트는 푸터에서 제거한다.

## 구현 설계

- 네 HTML 파일의 `<footer class="site-footer">` 내부 마크업을 동일하게 맞춘다.
- 심볼은 헤더에서 이미 사용하는 `/assets/app-icons/v2.0.0/fedshot911-brand-mark-48-v2.0.0.png`를 재사용한다.
- 심볼 링크에는 화면 낭독기를 위한 `aria-label="fedshot911 tennis home"`을 적용하고 이미지는 장식 요소로 처리한다.
- 공통 CSS `styles.v1.1.1.css`에 푸터 심볼 크기와 링크 정렬 규칙만 추가한다.
- 데스크톱과 모바일 모두 심볼은 좌측, 링크는 우측인 한 줄 배치를 유지한다.
- 기존 푸터 여백, 색상, 테니스공 커서 규칙은 유지한다.
- 공통 CSS 캐시 키는 새 날짜·시간 값으로 네 HTML에서 동일하게 갱신한다.

## 접근성 및 반응형

- 심볼 링크의 키보드 포커스가 보이도록 기존 링크 포커스 색상과 브라우저 기본 윤곽선을 유지한다.
- 작은 화면에서도 좌우 요소가 겹치지 않도록 푸터 링크 영역을 줄바꿈 가능한 한 묶음으로 유지하되, 푸터 자체는 한 줄로 배치한다.
- 심볼 이미지는 비율을 유지하고 과도하게 커지지 않도록 고정 크기를 적용한다.

## 검증

- 정적 검사로 네 HTML의 푸터 마크업이 동일한지 확인한다.
- 로컬 브라우저에서 네 페이지의 푸터 심볼, 링크 내용, 링크 목적지를 확인한다.
- 데스크톱과 모바일 화면에서 좌우 배치와 겹침 여부를 확인한다.
- 심볼 클릭 시 홈페이지로 이동하는지 확인한다.
- Privacy와 Contact 링크가 각 대상 페이지로 이동하는지 확인한다.
- 관련 콘솔 오류와 이미지 로드 오류가 없는지 확인한다.

## 변경 제외

- 헤더, 본문, 라켓 추천 기능, 개인정보 문구는 변경하지 않는다.
- 새 이미지나 새 라이브러리는 추가하지 않는다.
